Background

The Problem

  • Often an application needs to deployed to more than one cluster

    • Different Data Centers for Disaster Recovery

    • Different Data Centers for Geographical distribution

    • Different Data Centers for Load Management

  • OpenShift does not support Multi-Cluster Deployments

    • Neither does Kubernetes

  • Network Latency between clusters is always a concern

Solution Architecture

Networking Considerations

  • For networking we are going to assume the following requirements:

    • Each application will have a unique vanity URL
    • Applications should also have well-known cluster-specific URLs. It is ok if these URLs are available only internally (that is, they are not exposed to the internet).
    • We are willing to sacrifice a bit of flexibility in the URL naming, in favor of automation.
    • HTTPS must be supported.
    • Each application must be able to fail over individually.
